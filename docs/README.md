# docs ディレクトリについて
- rules/CORE.md
  → 開発時に必ず守る全プロジェクト共通のルール（唯一の正は `~/development/devkit/docs/rules/CORE.md`。このリポジトリの `docs/rules/CORE.md` は案内）
- rules/LOCAL.md
  → 開発時に必ず守るプロジェクト固有のルール
- REQUIREMENTS.md
  → ビジネス要件・機能要件の定義
- SPECIFICATIONS.md
  → 技術仕様・詳細設計の定義
- REVIEW.md
  → レビュー結果・テスト観点の記録
- CHANGELOG.md
  → バージョンごとの変更履歴
- HANDOFF.md
  → セッションをまたぐ際の引き継ぎ
- decisions/
  → 意思決定ログ（ADR）
